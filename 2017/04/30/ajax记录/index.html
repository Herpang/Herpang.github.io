AJAX记录

ajax以前也断断续续的学过很多次了，看了很多相关的博客，也简单封装过一次，这次系统的总结一下。

ajax是一种不需要刷新页面就可以和服务器进行数据交互的技术，其实就是可以异步发送http请求的技术，ajax的核心是XMLHttprequest对象，在这可以回忆下form表单是怎么和服务器交互的。
这里又涉及到了js的单线程机制，就是说js的运行是单线程的，s当我们提交form表单的时候，页面开始刷新，我们这时候就必须等待服务器响应完毕之后给用户返回成功与否，这时候用户做不了其他事情，很影响用户体验。
而ajax则可以异步的和服务器进行数据交换，就是说用户在提交数据的时候可以同时进行其他的操作，并不需要等待服务器响应完成。

一次偶然看到了一个这样的理解，可以将ajax技术想象成一个中间层，我们只要把请求交给中间层就可以了，剩下的就让中间层和服务器进行交互，这时候我们就可以做其他事情去了，交互完成之后中间层再返回给我们结果，我们再按照设置好的回调函数进行操作就可以了。

上面这种理解我觉得还挺贴切的。

ajax的核心是XMLHttpRequest对象（就是我们刚才说的中间层），现在基本上所以浏览器都支持。
下面说一下具体的实现
我们必须的要用到的几个方法

  • open();方法 他会启动一个针对目标地址的请求，调用的时候不会真正的发送，只是准备好。这个方法 接收3个参数
    第一个是请求类型 get post等
    第二个是请求的URL
    第三个是 是否异步发送 true false 默认是异步发送
    open("get","example.php",true);
  • send(); 该方法接收1个参数，即我们要发送的数据。
    执行这个方法才会真正的发送请求和数据
    我们一般数据量比较小的时候会把参数放到url的后面，拼接字符串
    一般发送post请求的时候才会给send方法里面放数据（注意，使用post请求的时候一定要在open方法和sned方法之间设置请求头 setRequestHeader(),这里我们后面说）
  • responseText 属性，这个属性里面存放着服务器发回来的数据（只要格式不是XML，都会放在这里），也就是说数据交互成功之后我们从这个属性里面就可以得到返回的数据了。
  • responseXML 当返回的数据格式是XML的时候，数据就会存在这里面（一般数据交互的时候用的都是json ）
  • status 响应的HTTP状态码 可以根据这个状态码来看是否出错，出错了该怎么调
  • statusText Http状态的说明
    一般来说，在我们接到服务器响应的时候第一步就是要检查status属性， 200表示一切正常，并成功返回，这个时候我们就可以从responseText里面拿数据了，
  • getResponseHeader()和 getALLResponseHeader()这两个方法是获取服务器响应的头部信息，说实话，这两个方法我还没用过，但是能看懂，第一个方法可以一个一个的查头部的字段，第二个是直接查询头部所有的字段，建议用第二个。

看到这里可能看烦了，咱下面边看代码边说。

第一步，我们先创建xhr对象

function creatXHR(){
if(window.XMLHttpRequest){
    return XMLHttpRequest();
    }
}else  {// IE6, IE5  
  xmlhttp=new ActiveXObject("Microsoft.XMLHTTP");  
}

其实就相当于 

var xhr = new XMLHttpRequest();

现在兼容性一般都没什么问题。

第二步，设置回调函数

xhr.onreadystatechange = function () {
    if(xhr.readyState === 4 && xhr.status === 200){ //这里其实应该判断的再准确点
        success(xhr.responseText);
    }else{
        error(xhr.readyState,xhr.status);
    }
}

这里有一点上边没说，就是readyState属性，这个属性表示我们发送的请求的状态，取值0-4，4表示完成，这个值每次变化都会触发onreadystatechange，所以我们检查onreadystatechange就可以了,当readyState和status都没问题之后我们就可以执行毁掉函数了。

第三步，open和send

xhr.open(type,url,async);
xhr.setRequestHeader("Content-type","application/x-www-form-urlencoded")
xhr.send(data);

然后基本上就差不多了，
