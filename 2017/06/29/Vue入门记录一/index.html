Vue入门记录一

发表于 2017-06-29

前言
Vue也用了有一段时间了，趁着今天bug改完的比较早，抓紧时间记录下一路以来学习Vue的一些感悟和一些基础知识。

我文笔很差，不建议看我这篇入门，移步官方文档或者各大神的博客吧
看官方文档入门Vue一般都没问题（Vue的官方文档写的很棒，在这里赞一个！ ）
先说Vue是什么，一个渐进式框架，mvvm库，核心是数据驱动和组件化。
（使用vue要转变以前操作dom的思想）
(mvvm即 view<—–>viewmodel<—–>model )

开始

写的比较随心所欲。。。
其实我们使用Vue的过程就是定义mvvm各组成部分的过程

就是我们的view层了
data：｛｝ 就是我们的model层了
new Vue（｛ el:#xxx; data:xxx ｝）就是我们的viewmodel层了，这个是mvvm的核心，就是我们的vue实例。
view层改变的时候 viewmodel层中有个dom listener会监听dom的改变并反馈到model层中
model层改变的时候 data binding工具也会实时的更新到view层中
这就是双向绑定
viewmodel层会处理其中复杂的逻辑，我们的主要操作也是在viewmodel层中
这里直接上大神博客里的截图了

vue实例接收一个选项对象，包括挂载元素（就是放到哪个元素下面），方法（就是我们要用到的自定义的一些方法），生命周期钩子函数（很重要的一部分，在实例创建的各个时期允许我们操作的地方）等

指令

介绍了一些基础之后 下面详细说说指令，也很重要
v-if 渲染指令，v-if = “xxx” 如果xxx可以是bool属性，也可以是一个返回bool值的表达式

v-show 指令和v-if指令用法差不多，但是v-if指令是决定要不要渲染到dom上，而v-show会始终
渲染到dom上，他只是决定display是否会显示
v-else 指令 配合上面的v-if 和v-show使用，如果v-if为真，就只执行v-if，
如果v-if为假，就执行v-else，执行v-else的时候和v-if的效果是一样的
v-bind指令 v-bind: class=”xxxx” 一般来说v-bind后面会接一个参数，
这个参数我们一般用HTML元素的属性，就比如说class,就像以前用classname操作样式的呢个思路
v-on指令用于给监听DOM事件，它的用语法和v-bind是类似的，例如监听元素的点击事件
草草收尾，主要用作记录。
