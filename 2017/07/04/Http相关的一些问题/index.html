一些基础

100-199 用于指定客户端应相应的某些动作。
200-299 用于表示请求成功。
300-399 用于已经移动的文件并且常被包含在定位头信息中指定新的地址信息。
400-499 用于指出客户端的错误。
500-599 用于支持服务器错误。

1,302状态码

第二次遇到302状态码了
302：

记得上一次遇到是前端发post请求给后端，后端打断点死活收不到请求，地址，格式，什么的都没问题，最后发现是一个学长加了一个拦截器把所有请求都拦截掉了，然后还忘了这回事儿了。。。
这次遇到302也是，在浏览器看请求的信息，发送的数据，地址都没问题，也是请求不到信息，由此发觉错误应该在后端，经过排查，因为项目要上线，老大加了个登录验证功能，没有登录的时候请求不到数据。
