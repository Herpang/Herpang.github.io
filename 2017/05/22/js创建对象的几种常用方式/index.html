js创建对象的几种常用方式

发表于 2017-05-22

1.工厂模式

很基础的一种方法，解决了创建多个相似对象的问题，主要是抽象了创建对象的过程，将对象的属性作为参数传入。

2.构造函数模式

和工厂模式相比，构造函数的模式并没有显式的创建对象，构造函数直接将属性和方法赋给了this对象，也没用return对象。 注意，构造函数名我们默认首字母大写


构造函数的过程

1，会首先创建一个对象
2，将构造函数的作用域赋给新的对象（这样this就指向了这个新对象）
3，执行构造函数里面的代码
4，返回新对象

我们创建自定义的构造函数时，意味着可以把他的实例标识为一种特定的类型，用instanceof可以检测出来

构造函数和普通函数的区别，唯一的区别就是调用的方式不同，

实际上，任何函数用new操作符调用，都是构造函数，而构造函数不用new操作符调用，和普通的函数没什么两样。

构造函数存在的一些问题

主要是构造函数创建的方法，由构造函数创建的方法是在每个实例都实现一遍的，也就是说不同的实例之间互不影响，不能共享，在我们不需要多个实例共享方法的时候不是问题。

3.原型模式

说到原型模式肯定离不开原型继承和原型链(我有单独写一篇，可以去看一下)
原型模式主要利用了prototype这个属性，我们每创建一个函数，都会为该函数创建一个原型，函数本身都会有一个prototype属性，指向该原型，原型本身也会有一个constructor属性，指向原函数。

这样，根据原型链的方向，每个Person函数的实例都能从Person函数的原型上读到所有的属性和方法，并且这些方法是共享的。

4.组合使用构造函数和原型模式

这种方式是我们很常用的一种方式，在构造函数里面定义实例的属性，在原型模式里面定义方法和共有的属性。这样就可以共享着方法的使用，每个实例也会有一个属于自己的属性的副本，还可以向构造函数传参，最大限度的节省了内存。

5.寄生构造函数模式

寄生构造函数模式几乎和工厂模式一模一样，唯一的一点区别是，寄生构造函数调用的时候是用new操作符调用的，而工厂模式是直接调用的。仅此而已。
寄生构造函数的模式在一些特定的情况下还是很好用的，比如说，我们想给一个数组添加一个自定义的方法，由于我们不能直接修改数组的构造函数，这时候寄生构造函数就可以派上用场了，具体的看代码！

6.稳妥构造函数模式

稳妥构造函数，就像他的名字，很稳妥，在什么模式下面都可以用，比如说禁止使用this和new的时候，这种模式很适合在一些安全环境中使用（具体的使用场景我还没经历过，没能深刻领悟）
